# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Content generator + Hugo site for **Drus**, a tabletop RPG magic system wiki (Czech language). Spell lists live in plain-text/JSON data files; Python scripts render them into Hugo markdown content; Hugo (theme PaperMod) builds the static site; GitHub Actions deploys it to the `github-pages` branch.

## Build / run

```bash
python3 main.py          # regenerates DrusMagie/content/magic/*.md from lists/*.txt + secrets
cd DrusMagie && hugo      # build the static site (serve with: hugo server)
```

`main.py` just calls `magic.main()` then `counter_magic.main()` — always regenerate content this way before building Hugo; never hand-edit files under `DrusMagie/content/magic/`, they are generated output.

There is no test suite and no linter configured. `DrusMagie/themes/PaperMod` is a git submodule (`git submodule update --init` if missing).

## Editing secrets

`lists/secrets.json` stores base32-encoded plaintext ("secret" spells hidden from the public lists). Never hand-edit the encoded blobs — use:

```bash
python3 edit_secrets.py   # decodes each entry to a temp file, opens it in `code --wait`, re-encodes on save
```

## Architecture

**Data flow:** `lists/*.txt` (per-school spell definitions, Czech, tab-indented custom format) + `lists/kontramagie_layout.json` (skill-tree node positions) → parsed by `common.py` → rendered to markdown/HTML by `magic.py` and `counter_magic.py` → written into `DrusMagie/content/magic/*.md` → built by Hugo.

**`common.py`** — shared parsing/model layer:
- `parse_magic_file` / `parse_magic_file_s` parse the custom spell-list text format: top-level tier headers (`základní:`, etc.), tab-indented spell headers (`name -- tag -- tag`), and tab-tab-indented description bullets (`\t\t- ...`).
- `Spell` (NamedTuple) and `SpellCategory`/`MagicSchool` are the resulting data model.
- `parse_secrets` decodes base32 entries from `lists/secrets.json` by key.
- `MagicSchoolTemplate` fills `templates/school_of_magic_header.md` (the Hugo page front-matter + `$CONTENTS` placeholder) for every generated school page.

**`magic.py`** — generates the six ordinary magic school pages (`cas`, `prostor`, `zivot`, `smrt`, `hmota`, `mysl`):
- `TooltipTemplate.spell_tooltips` is the single source of truth mapping a spell tag (Czech word) to an emoji icon — add new tags here when adding new tag keywords in the lists.
- `SpellLineTemplate.glow_mapping` maps tag sets to a CSS glow class for the spell's tooltip highlight color; first matching entry wins, falls back to `neutral_glow`.
- `filter_bar_to_html` renders the per-school tag filter buttons (`MagicSettings.filter_tags` per school) consumed by client-side JS in the theme's custom CSS/JS.
- Each school merges its public `lists/<name>.txt` spells with the decoded secret spells of the same name from `secrets.json`, secret categories get a `secret_header` CSS class and secret spell rows get `secret_spell` class.

**`counter_magic.py`** — generates the single "Kontramágové" (counter-magic) skill-tree page, which is structurally different from the school pages:
- Renders an SVG skill tree: nodes positioned by percentage coordinates from `lists/kontramagie_layout.json` (`pos_x`/`pos_y`, `category`, `depends_on` edges with optional `source_angle`/`target_angle` for bezier curve routing), circles/labels via `CounterMagicHTML`, connecting arrows/divider lines via `CounterMagicSVG`.
- `CounterMagicConfig` holds all magic-number layout constants (radii, divider row positions, per-category colors) — tune here, not inline.
- Also merges in secret nodes/spells from `secrets.json` (`kontramagie_layout` + `kontramagie` keys).
- `_create_school_arc_ring` draws the small per-node arc-ring of school icons for nodes marked `learned_per_school` in the layout JSON.

**Templates (`templates/`)** are plain-text files with `$PLACEHOLDER` tokens replaced via `str.replace` (no templating engine) — keep new templates consistent with this convention.

**Hugo site (`DrusMagie/`)** — standard Hugo + PaperMod theme site. Custom styling lives in `DrusMagie/assets/css/extended/custom.css` and `counter_magic_styling.css` (referenced via `customCSS` in `hugo.yaml`); custom footer/shortcode partials in `DrusMagie/layouts/`. `DrusMagie/content/magic/*.md` is fully regenerated by the Python scripts — do not edit by hand.

## CI

`.github/workflows/build.yml` on push to `master`: runs `python3 main.py`, builds with `hugo`, publishes `DrusMagie/public` to the `github-pages` branch. Any change to spell lists or generator scripts needs no manual regeneration step for deploy — CI does it — but regenerate locally when previewing.
